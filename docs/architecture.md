# Yunyez 架构设计

## 1. 项目目标
打造一个可定制的电子宠物服务端系统，支持：
- 多设备接入（资源受限型嵌入式设备）
- 实时语音交互（ASR → AI → TTS）
- 图像感知与行为决策
- 高可靠、低延迟通信
- 模块化扩展（未来支持插件、技能市场等）

## 2. 核心原则
- **设备轻量化**：
- - 初期：设备只负责采集 + 执行，智能在服务端
- **异步解耦**：服务间通信优先使用消息队列（Kafka/RabbitMQ）
- **协议统一**：所有设备通信走 MQTT（主）或 WebSocket（调试）
- **可观测性**：全链路日志 + SQL 记录 + 指标埋点

## 4. 技术栈
- **语言**：Go（高并发、内存可控）
- **通信**：MQTT（EMQX）、WebSocket、UDP（图像传输）
- **存储**：PostgreSQL（设备元数据）、Redis（会话/缓存）、MongoDB（日志/非结构化）
- **消息队列**：Kafka（高吞吐） / RabbitMQ（简单场景）
- **AI**：阿里通义千问（Qwen）API（初期）
